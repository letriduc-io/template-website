Overview

Meteor - a responsive and multipurpose admin dashboard template, based on latest Twitter Bootstrap framework. It's super flexible and user friendly, as it's compatible with any device. We put a lot of time and effort to create such a template, with many kinds of different layout variations and yet 3 "layers", as many others are coming soon for the next updates. They can be changed just by changing a line of code, so you haven't to worry about choosing another one! This makes Meteor an easily customizable tool for anyone.

If you have any questions that are beyond the scope of this help file, please feel free to email via our user page contact form here. Thanks so much!

Customization

All theme CSS files are located in Source/../assets/css folder. For your CSS customization you can use Source/../assets/css/custom.css. Keep your custom CSS separate it makes more easier when updating the theme in future.

All Javascript files are located in Source/../assets/js/ folder. Also Page-level Javascript files are located in source/assets/js/pages/ folder and each page has its own Javascript file. For your Javascripts you can use Source/assets/js/custom.js. Keep your custom Javascript separate it makes more easier when updating the theme in future.

Fonts

This template uses Arial font.

Meteor also uses Fontawesome, Glyphicons, Weather icons and Simple Line Font Icon packs. Font Icons must be loaded in HTML Head before theme css files.

Themes & Layouts

Meteor comes with 3 layouts, we call them "layers": dark, light and material. You can check them by clicking on button before full screen and after collapse menu buttons.

Once you've chosen one of the "layers", you must include the layer code in HTML Head. For example, if you'd like to have a light one, you must write the following code:

<link href="assets/css/layers/light-layer.css" class="theme-color" rel="stylesheet" type="text/css"/>

If you'd like to have a dark one, then don't touch anything. It's set by the default.

Meteor comes with many flexible and functional layout options. It lets you to create the most relevant layout for your application. In Meteor UI Framework layouts are selected by adding classes on the HTML body tag or/and exactly on that element which must be changed. You can choose layout options very easily by playing with them in themes & layouts settings dropdown menu, which is located on left side of topmenu.

By default Meteor uses Static Header Mode. If you don't want to use Static Header Mode, then adding class page-header-fixed to body must help.

To use Fixed Sidebar Mode you must add page-sidebar-fixed class to HTML body. By default Meteor uses Static Sidebar Mode.

To use Horizontal Bar Mode you must add page-horizontal-bar class to HTML body, add horizontal-bar class to .sidebar and remove page-sidebar class from it. By default Meteor doesn't use Horizontal Bar Mode.

If you want to use Collapsed Sidebar Mode, then you just have to add a small-sidebar class to HTML body. Also, there is a special button in the topmenu which toggles Collapsed Sidebar Mode. By default Meteor is not using Collapsed Sidebar Mode.

Meteor comes with compact/compactALT menu modes. To use compactALT menu, you must remove compact-menu class from HTML body. compact menu is set by default.

Meteor also includes Accordion/Hover menu modes. To use hover menu you must add hover-menu class to HTML body. Accordion mode is set by default.

Meteor also includes Boxed Layout Mode. To use it you must add container class to .page-content. By default Meteor doesn't use this layout option.

Note! Some Layout options aren't compatible together. You can check it by changing layout options from themes & layouts settings dropdown menu.
